eclipse中hadoop环境

hdfs配置

首先要下载hadoop-eclipse-plugin-**.jar，至于在哪里下，google一下。下载对应版本的插件后，放到eclipse的plugins目录下，然后打开eclipse：

打开window–>preferences ,配置Hadoop MapReduce的安装路径；

打开Window–>Show View–>Other 窗口，选择 MapReducer Locations；

此时在底部的视图中应该会出现mapreduce locations，点击右上角蓝色小象新建一个hadoop地址，并配置mapreduce master和hdfs master，其中：

  • MapReduce Master：与$HADOOP_DIRCONF/mapred-site.xml配置保持一致，注意，host和port都要保持一致（一般是9001）；
  • HDFS Master：与$HADOOP_DIRCONF/core-site.xml配置保持一致（一般是9000）
  • User Name：登录hadoop用户名，可以随意填写

此时，点击eclipse右上角的蓝色小加号按钮，选择添加mapreduce到project explore视图中，打开你的hadoop，就可以从eclipse中访问hdfs了。

mapreduce

直接创建mapreduce工程，直接跑就行。需要注意的是，读文件时以hdfs://master:9000/….为开头读取；写文件时必须加上，如果目标路径不存在则写，不然会报错。其他还没有发现需要特别注意的事情。

spark搭建过程

我的虚拟机ubuntu，步骤是

  • 下载sacla，解压到目标文件夹，配置/etc/profile，Scala不用配置path。
  • 下载spark，解压到目标文件夹，配置/etc/profile，需要配置path。
  • 复制上述两个文件夹到子节点中，并一样的配置profile。

检查是否成功：
cd到spark的sbin下，./start-all.sh，然后jps查看是否有一个worker的进程，子节点和主节点都有的话，就成功了。

eclipse中spark-scala的环境

在这里找到eclipse版本对应的scala插件。将地址复制进
help -> install new …里面打开，安装其中的scala IDE for eclipse 和 ** development support两个，然后就可以创建scala project了：

  • project -> proporties
  • 设置scala compiler->fix 2.11.6 - 2.11.8都可以
  • java build path -> libraries -> add extern jars中添加spark根目录下jars文件夹内的所有jar包

其他还有几点需要注意：

  • 调试程序时只能使用本地的文件作为输入，写起来比较方便，可以随时跑起来。spark conf必须要设置master
  • 在hdfs中跑程序，需要先打包成.jar文件利用spark-submit命令提交到spark执行，可以在localhost:4000中查看运行的任务。
