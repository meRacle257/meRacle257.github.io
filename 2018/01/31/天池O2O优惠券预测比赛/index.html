废话不说，开始回顾。

赛题

题目大致意思是，给出了用户在2017年1月到6月的线上线下消费情况的数据，预测用户在七月份的优惠券使用情况。具体来说，给出了几个表，包括优惠券使用情况和消费情况等等。赛题内容：戳这里

方案设计

方案大体分为特征工程和训练模型两个部分。

特征工程

数据分析

整体来看，线下数据比较多，应该是线上数据的10倍左右，但是题目又是预测用户线上的优惠券使用情况，刚开始是很懵圈的，但是后来分析了一下，线下数据也许能从侧面反映用户的购买力度，虽然可能是和结果是负相关（因为线下越活跃线上可能就需求没那么大），所以就针对线上的数据提取了一些反映用户活跃度的特征，但是总体还是使用相关度很大的线上数据。

数据集划分

数据集总的给了6个月的用户信息，根据六个月预测十五天内的优惠券使用情况。实际上，用户最终使用情况可能仅仅和最近的行为关系密切，时间久的就并没有那么的游泳了。基于这样的想法，采用了滑动窗口方法，我分的很细，分成了五个数据集，具体来说就是利用1、2月的数据+3月上旬的优惠券使用；2、3月的数据+4月上旬的优惠券使用等等。依次形成了五个数据集。

特征选择

按照现学现卖的套路，学着各种前几名的前辈们的方法，尽量去分析可能相关的几个主题，然后再依据这几个主题去找他们的相关关系，基本上是按照几个一组的形式去添加属性，在测试集上的效果也是稳步提升。值得一提的是，线上的数据加上之后，果然是发生了奇效，线上测试的AUC提升了近1%（最为重要的百分之一），直接从100多跳到了11名。关于具体的特征有很多，在Github上写的很清楚，这里就是简单的列一下几个主体。

  1. 用户线下相关的特征（唯一线下特征集）
  2. 用户线上相关的特征
  3. 商家相关的特征
  4. 用户-商家交互特征
  5. 优惠券相关的特征
  6. 其它特征

模型训练

XGBoost安装

XGBoost全名eXtreme Gradient Boosting，是在各个数据比赛中使用效果极佳的一个分类模型。XGBoost前身GBDT，在前身采用CART（回归树）基分类器的基础上，比GBDT不同的是，增加了对二阶导数的利用，并且增加了多线程处理的方法，极大提高了运行速度。关于原理和使用说明戳这里，这个是别人的连接，个人很喜欢这个模型，有机会一定会研究一下，写一下自己的理解。

这里主要记录一下在Mac下安装的XGBoost的步骤：

  1. 因为mac默认gcc是clang，需要安装gcc5，若已安装过请忽略：brew install gcc5
  2. 修改默认编译器（等号左右不加空格）：export CC=gcc-5 export CXX=g -5
  3. pip安装xgboost：pip3 install xgboost

模型选择

模型上本来用的是单个模型XGB，但是效果始终上不去，在本地测试集上效果很好，线上却一直没有突破。后来加上了GBDT，本地效果一般，但是结合了XGB后线上表现特别好。这让我一度认为这都是玄学，后来仔细的分析了一下。XGB本身这个模型就是有过拟合的倾向，而GBDT我给的迭代次数也很少，所以最终可能产生的震动比较大，一定程度上缓和了XGB造成的过拟合，所以才有了泛化能力的提升。

参数模块，基本上都是按照这里的一篇不错的博文来搞的。XGB调参指南下面贴出来最终的参数。

params={'booster':'gbtree',
        'objective': 'rank:pairwise',
        'eval_metric':'auc',
       'gamma':0.1,
       'min_child_weight':1.1,
        'max_depth':5,
       'lambda':10,
       'subsample':0.65,
       'colsample_bytree':0.65,
       'colsample_bylevel':0.65,
       'eta': 0.01,
       'tree_method':'exact',
       'seed':0,
       'nthread':12
}

总结

经过这个比赛，我发现数据挖掘比赛的精髓就是特征工程部分，而且最难做的也就是特征工程部分，一般来说给出的数据都不需要很麻烦的数据清洗，模型来说，大家基本上用的也都差不多，不是神经网络就是提升方法，差距就在特征工程这块，做的好了能表现的非常明显。

学习到了不少东西，没法一个一个说了，感觉很爽的就是pandas的一些用法，真的是好用！还有就是熟悉项目实战过程、模型选择和调参等等各个方面的进步。加油加油，砥砺前行，过年回家好好学数学、算法，然后再参加一个比赛。
